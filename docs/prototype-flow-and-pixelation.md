# FrameGrid 原型流程确认

## 1. 用户使用流程与页面布局

这个原型是一个本地网页工作台，用户通过浏览器地址访问。

### 首页：项目列表

用户打开地址后首先看到项目列表。

主要区域：

- 顶部：FrameGrid 标题、当前生成资产根目录提示。
- 左侧或主体：项目卡片列表。
- 右侧：创建项目面板。

主要按钮：

- `创建项目`
- 项目卡片上的 `进入项目`

点击 `创建项目` 后，系统会在项目根目录下的专门目录中创建一个项目文件夹，例如：

```text
D:\FrameGrid\generated-projects\project_xxx\
```

### 项目页：角色列表

进入某个项目后，页面展示这个项目下的所有角色。

主要区域：

- 顶部：项目名称、返回项目列表按钮。
- 主体：角色卡片列表。
- 侧边面板：创建角色表单。

主要按钮：

- `返回项目`
- `创建角色`
- 角色卡片上的 `进入角色`

点击 `创建角色` 后，用户输入角色名称和角色提示词。系统会先生成透明背景的角色三视图：

创建角色时还需要选择目标像素尺寸，例如：

```text
32x32
48x48
64x64
96x96
128x128
```

这个尺寸会写入角色配置，并进入三视图和动作帧的提示词。生成提示会要求模型像直接在目标低分辨率画布上绘制，而不是先生成复杂高分辨率插画再后处理。

```text
front.png
side.png
top.png
```

文件保存到：

```text
D:\FrameGrid\generated-projects\project_xxx\characters\character_xxx\views\
```

### 角色页：三视图与动作生成

进入角色后，页面展示角色专属工作区。

主要区域：

- 顶部：角色名称、返回项目按钮。
- 三视图区：正视图、侧视图、顶视图。
- 动作区：已生成动作列表。
- 右侧面板：生成动作、像素规整。

主要按钮：

- `返回项目`
- `生成动作`
- `像素规整`
- 动作卡片上的 `查看动作`

用户点击 `生成动作` 后，输入动作名称、动作提示词、帧数和 FPS。系统会根据角色三视图和动作提示词生成一组透明背景动作帧。

生成动作前，系统会先生成动作分镜 `storyboard.json`。分镜会描述每一帧的：

- 方向与镜头，例如 `side view facing left`。
- 腿部和脚的位置。
- 手臂摆动。
- 身体重心和上下起伏。
- 与相邻帧必须不同的姿势变化。

之后每一帧生图都会携带对应的分镜说明。比如 `walk left` 会明确要求双腿前后交替、手臂反向摆动、身体有 1-2 个逻辑像素的上下起伏。

动作文件保存到：

```text
D:\FrameGrid\generated-projects\project_xxx\characters\character_xxx\actions\walk\
  frames\
    frame_001.png
    frame_002.png
    frame_003.png
  spritesheet.png
  preview.gif
  action.json
```

### 动作页：帧预览与导出入口

点击动作后进入动作详情。

主要区域：

- 顶部：动作名称、返回角色按钮。
- 主体：动画预览、spritesheet、单帧列表。
- 侧边面板：重新生成、像素规整、导出入口。

主要按钮：

- `返回角色`
- `像素规整`
- `导出帧`
- `导出 spritesheet`

第一版原型的核心路径是：

```text
创建项目 -> 进入项目 -> 创建角色 -> 生成三视图 -> 进入角色 -> 生成动作 -> 像素规整 -> 查看动作
```

## 2. 骨骼与动作模板流程

后续动作生成不再只依赖文字分镜，而是引入骨骼配置和动作模板。

### 骨骼配置

骨骼配置定义角色有哪些节点，以及节点之间的父子连接关系。

当前原型内置：

```text
humanoid_basic
humanoid_weapon
humanoid_cape
```

创建角色时选择骨骼配置，角色会保存这套配置的完整副本。创建表单和角色详情页都会显示骨骼预览，方便确认节点和连接关系。后续动作模板会基于角色骨骼检查和生成。

### 动作模板

动作模板是一组已经摆好的 pose。每个 pose 同时包含：

```text
pose JSON：每个骨骼点的位置
pose guide PNG：把 pose 渲染出来的骨骼参考图
```

生成动作时，系统不再临时让 AI 理解动作如何运动，而是：

```text
选择动作模板
-> 读取模板中的多帧 pose
-> 渲染每帧 pose guide
-> 把角色三视图 + 当前帧 pose guide 发给 AI
-> AI 生成最终角色动作帧
```

角色详情页会显示动作模板预览：模板卡片展示一排小 pose，生成动作表单会展示当前选中模板的完整 pose 时间轴。用户可以先检查动作幅度、脚步交替和武器/披风节点，再决定是否生成最终图片帧。

### AI 生成动作模板

如果没有合适的动作模板，用户可以输入动作描述，让系统根据当前角色骨骼生成一套新的动作模板草稿。

生成结果会保存到：

```text
generated-projects/<project>/action-templates/<template_id>/
  template.json
  guides/
    frame_001_pose.png
    frame_002_pose.png
```

这套模板可以继续被角色动作生成流程使用。后续会把模板编辑升级成可拖拽的骨骼编辑器。

## 3. 像素规整方法建议

原始想法是：把图片分成几乘几的格子，然后用 OpenCV 识别每个格子中出现最多的颜色，再把整个格子填成这个颜色。

这个方法可行，本质上是块状像素化。不过 AI 生成图通常带有抗锯齿、渐变和细碎阴影，直接统计“出现最多的颜色”容易不稳定，因为相近颜色可能被拆成很多个不同 RGB 值。

更稳的第一版流程建议是：

```text
原图
 -> 保留透明通道
 -> 按目标逻辑像素尺寸下采样，例如 32x32 或 64x64
 -> 使用颜色量化限制调色板数量
 -> 用最近邻放大回预览尺寸
 -> 输出规整后的像素图
```

具体实现上可以用：

- OpenCV `INTER_AREA` 做下采样，让每个逻辑像素综合原图局部颜色。
- K-means 或 median cut 做颜色量化，例如限制到 16、24、32 色。
- OpenCV `INTER_NEAREST` 或 Pillow 最近邻放大，确保边缘是硬像素。
- 对动画帧使用同一个画布尺寸、同一个调色板策略和同一套锚点规则，避免播放时闪烁或抖动。

因此，原型中保留原有兜底方案：

```text
OpenCV 下采样 + 调色板量化 + 最近邻放大
```

同时新增 `perfect-pixel` 模式，接入 `theamusing/perfectPixel`：

```text
自动检测像素网格
-> 根据边缘强度微调网格线
-> 每个格子用 center / median / majority 采样出代表色
-> 保留透明通道并阈值化
-> 最近邻放大回预览尺寸
```

当前角色页的 `完美像素化` 按钮默认使用 `perfectPixel 自动网格`。如果自动检测失败，系统会回退到原来的 OpenCV 目标尺寸规整，避免生成流程中断。也可以手动选择 `perfectPixel 指定尺寸` 或 `OpenCV 目标尺寸`。
