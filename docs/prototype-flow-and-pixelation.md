# FrameGrid 原型流程确认

## 1. 用户使用流程与页面布局

这个原型是一个本地网页工作台，用户通过浏览器地址访问。

### 首页：项目列表

用户打开地址后首先看到项目列表。

主要区域：

- 顶部：FrameGrid 标题、当前生成资产根目录提示。
- 左侧或主体：项目卡片列表。
- 右侧：创建项目面板。

主要按钮：

- `创建项目`
- 项目卡片上的 `进入项目`

点击 `创建项目` 后，系统会在项目根目录下的专门目录中创建一个项目文件夹，例如：

```text
D:\FrameGrid\generated-projects\project_xxx\
```

### 项目页：角色列表

进入某个项目后，页面展示这个项目下的所有角色。

主要区域：

- 顶部：项目名称、返回项目列表按钮。
- 主体：角色卡片列表。
- 侧边面板：创建角色表单。

主要按钮：

- `返回项目`
- `创建角色`
- 角色卡片上的 `进入角色`

点击 `创建角色` 后，用户输入角色名称和角色提示词。系统会先生成透明背景的角色三视图：

创建角色时还需要选择目标像素尺寸，例如：

```text
32x32
48x48
64x64
96x96
128x128
```

这个尺寸会写入角色配置，并进入三视图和动作帧的提示词。生成提示会要求模型像直接在目标低分辨率画布上绘制，而不是先生成复杂高分辨率插画再后处理。

```text
front.png
side.png
top.png
```

文件保存到：

```text
D:\FrameGrid\generated-projects\project_xxx\characters\character_xxx\views\
```

### 角色页：三视图与动作生成

进入角色后，页面展示角色专属工作区。

主要区域：

- 顶部：角色名称、返回项目按钮。
- 三视图区：正视图、侧视图、顶视图。
- 动作区：已生成动作列表。
- 右侧面板：生成动作、像素规整。

主要按钮：

- `返回项目`
- `生成动作`
- `像素规整`
- 动作卡片上的 `查看动作`

用户点击 `生成动作` 后，输入动作名称、动作提示词、帧数和 FPS。系统会根据角色三视图和动作提示词生成一组透明背景动作帧。

生成动作前，系统会先生成动作分镜 `storyboard.json`。分镜会描述每一帧的：

- 方向与镜头，例如 `side view facing left`。
- 腿部和脚的位置。
- 手臂摆动。
- 身体重心和上下起伏。
- 与相邻帧必须不同的姿势变化。

之后每一帧生图都会携带对应的分镜说明。比如 `walk left` 会明确要求双腿前后交替、手臂反向摆动、身体有 1-2 个逻辑像素的上下起伏。

动作文件保存到：

```text
D:\FrameGrid\generated-projects\project_xxx\characters\character_xxx\actions\walk\
  frames\
    frame_001.png
    frame_002.png
    frame_003.png
  spritesheet.png
  preview.gif
  action.json
```

### 动作页：帧预览与导出入口

点击动作后进入动作详情。

主要区域：

- 顶部：动作名称、返回角色按钮。
- 主体：动画预览、spritesheet、单帧列表。
- 侧边面板：重新生成、像素规整、导出入口。

主要按钮：

- `返回角色`
- `像素规整`
- `导出帧`
- `导出 spritesheet`

第一版原型的核心路径是：

```text
创建项目 -> 进入项目 -> 创建角色 -> 生成三视图 -> 进入角色 -> 生成动作 -> 像素规整 -> 查看动作
```

## 2. 像素规整方法建议

原始想法是：把图片分成几乘几的格子，然后用 OpenCV 识别每个格子中出现最多的颜色，再把整个格子填成这个颜色。

这个方法可行，本质上是块状像素化。不过 AI 生成图通常带有抗锯齿、渐变和细碎阴影，直接统计“出现最多的颜色”容易不稳定，因为相近颜色可能被拆成很多个不同 RGB 值。

更稳的第一版流程建议是：

```text
原图
 -> 保留透明通道
 -> 按目标逻辑像素尺寸下采样，例如 32x32 或 64x64
 -> 使用颜色量化限制调色板数量
 -> 用最近邻放大回预览尺寸
 -> 输出规整后的像素图
```

具体实现上可以用：

- OpenCV `INTER_AREA` 做下采样，让每个逻辑像素综合原图局部颜色。
- K-means 或 median cut 做颜色量化，例如限制到 16、24、32 色。
- OpenCV `INTER_NEAREST` 或 Pillow 最近邻放大，确保边缘是硬像素。
- 对动画帧使用同一个画布尺寸、同一个调色板策略和同一套锚点规则，避免播放时闪烁或抖动。

因此，原型中采用：

```text
OpenCV 下采样 + 调色板量化 + 最近邻放大
```

后续如果需要更接近“格子内主色填充”的严格算法，可以在规整参数里增加 `dominant-cell` 模式，与当前 `area-quantize` 模式并存。
